现在这版 qiankui.md 核心思想对了：已经把 253/254 启动阈值 和 157 稳态拟合截距 分开了，这个是正确的。

但里面有一个明显数学错误：

ff_abs = 157 + 18.3 * speed
所以：

speed=10 -> 157 + 18.3*10 = 340
speed=20 -> 157 + 18.3*20 = 523
它表里写成：

10 -> 254
20 -> 254
这是错的。

254 下限只有在：

157 + 18.3 * speed < 254
时才生效，也就是：

speed < (254 - 157) / 18.3 ≈ 5.3 pulse/20ms
所以只有目标速度低于大约 5.3 时，才会被强行抬到 254。常用速度 10、20、30 都不会触发下限。

正确对照表应该是：

speed 0  -> 0
speed 10 -> 340
speed 20 -> 523
speed 30 -> 706
speed 40 -> 889
speed 50 -> 1072
speed 60 -> 1255
其他地方基本对：

稳态公式：PWM = 157 + 18.3 * speed
启动下限：254
trust = feedforward + pid_out

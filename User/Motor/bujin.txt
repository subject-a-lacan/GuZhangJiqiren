张大头 Emm42_V5.0 / ZDT_X42_V1.2 42闭环步进电机串口控制整理
================================================================

用途
----
本文是给 Claude Code agent / STM32 固件开发用的协议速查。
目标是让单片机通过 UART/RS232/RS485 控制张大头 42 闭环步进电机。

原始资料在 `User/Motor/bujindianji.txt`，该文件包含网页 UI 和大量背景说明；
实现时优先看本文。除非需要查冷门参数，否则不要直接从原文里扒命令。


一、上电和菜单前置条件
----------------------
1. 电机驱动型号：Emm42_V5.0，硬件框架 ZDT_X42_V1.2。
2. 第一次使用必须完成编码器校准，否则电机可能提示 `Not Cal`。
3. 驱动板菜单 `P_Serial` 必须设为 `UART_FUN`，串口/RS232/RS485 控制才有效。
4. 默认串口波特率：115200。
5. 串口助手/MCU 发送时使用“十六进制字节流”，不要追加换行。
6. 默认设备地址 `ID_Addr = 0x01`；`0x00` 是广播地址，可让多个电机同时接收命令。
7. 默认校验方式 `Checksum = 0x6B`，即每条命令最后一个字节固定填 `0x6B`。
8. 默认控制命令应答 `Response = Receive`，即动作类命令收到后返回确认帧。


二、通用帧格式
--------------

### 1. 命令帧

通用格式：

```
[地址] [功能码] [参数...] [校验字节]
```

字段说明：

- `地址`：1 字节。默认 `0x01`；`0x00` 为广播地址。
- `功能码`：1 字节，例如速度模式 `0xF6`、位置模式 `0xFD`。
- `参数`：0 个或多个字节，按各命令定义排列。
- `校验字节`：默认固定 `0x6B`。

### 2. 字节序

多字节数值全部按高字节在前（big-endian）发送。

示例：

- `1500 RPM = 0x05DC`，发送 `05 DC`。
- `32000 脉冲 = 0x00007D00`，发送 `00 00 7D 00`。

### 3. 可选校验方式

实际比赛项目建议保持默认 `0x6B`，最省事。

- 固定校验：最后一个字节固定 `0x6B`。
- XOR：最后一个字节为前面所有字节异或值，例如 `01 06 45 42` 中 `42 = 01 ^ 06 ^ 45`。
- CRC-8：最后一个字节为前面所有字节的 CRC-8。

本文所有命令都按默认 `0x6B` 写。


三、返回帧和状态码
------------------

### 1. 动作/配置命令的常见返回

成功：

```
[地址] [功能码] 02 6B
```

条件不满足：

```
[地址] [功能码] E2 6B
```

常见原因：

- 电机未使能。
- 已触发堵转保护。
- 回零相关命令参数不满足条件。

错误命令：

```
[地址] 00 EE 6B
```

### 2. 位置模式到位返回

如果菜单 `Response` 设置为 `Reached`、`Both` 或 `Other`，位置模式到位后可能返回：

```
[地址] FD 9F 6B
```

默认 `Receive` 只确认收到命令，不主动返回到位帧。


四、参数编码约定
----------------

### 1. 方向 `dir`

- `0x00`：CW
- `0x01`：CCW

注意：资料示例中 `CCW` 常被描述为负方向。实际小车机构中正反方向需要实测决定。

### 2. 速度 `speed_rpm`

- 单位：RPM。
- 2 字节无符号数，高字节在前。
- 默认最小分辨率 1 RPM。
- 若启用 `S_Vel_IS`，输入速度会缩小 10 倍：发送 `1` 实际为 `0.1 RPM`。

### 3. 加速度 `acc`

- 1 字节，范围 `0x00` 到 `0xFF`。
- `0x00`：不使用曲线加减速，直接按设定速度运行。
- 非 0：使用曲线加减速。
- 资料给出的速度变化关系：每增加 1 RPM 的时间约为 `(256 - acc) * 50us`。

### 4. 多机同步标志 `sync`

- `0x00`：立即执行。
- `0x01`：先缓存命令，等待“多机同步运动”命令后再执行。

### 5. 保存标志 `save`

- `0x00`：不保存，断电丢失。
- `0x01`：保存到驱动器。

### 6. 位置/角度和脉冲数

位置模式命令使用“脉冲数”，不是直接发送角度。

资料示例说明：

- 在 16 细分下，`3200` 个脉冲 = 电机转 1 圈 = `360°`。
- 因此 16 细分时：`pulse = angle_deg / 360 * 3200`。
- 例如 `10 圈 = 3600° = 32000` 脉冲 = `0x00007D00`。

如果修改细分，单圈脉冲数也会变化。普通 1.8° 步进电机通常：

```
pulse_per_rev = 200 * microstep
```

其中 `microstep = 16` 时，`pulse_per_rev = 3200`。


五、核心动作命令
----------------

### 1. 电机使能控制

功能：使能或失能电机。

帧格式：

```
[addr] F3 AB [enable] [sync] 6B
```

参数：

- `enable = 0x01`：使能。
- `enable = 0x00`：失能。
- `sync = 0x00`：立即执行；`0x01`：等待同步触发。

示例：

```
01 F3 AB 01 00 6B    // 1号电机使能
01 F3 AB 00 00 6B    // 1号电机失能
```

成功返回：

```
01 F3 02 6B
```

### 2. 速度模式控制

功能：让电机按指定方向和速度连续转动。

帧格式：

```
[addr] F6 [dir] [speed_H] [speed_L] [acc] [sync] 6B
```

参数：

- `dir`：`00=CW`，`01=CCW`。
- `speed`：2 字节 RPM，高字节在前。
- `acc`：加速度档位，`00` 表示无加减速。
- `sync`：多机同步标志。

示例：

```
01 F6 01 05 DC 0A 00 6B
```

含义：

- 地址 `01`。
- 方向 `CCW`。
- 速度 `0x05DC = 1500 RPM`。
- 加速度档位 `0x0A = 10`。
- 不启用同步，立即执行。

成功返回：

```
01 F6 02 6B
```

使用建议：

- 小车只需要让步进电机连续转时，用该命令。
- 需要停车时用“立即停止”或发送速度为 0 的策略；资料明确支持“立即停止”。

### 3. 位置模式控制

功能：让电机转到指定相对/绝对位置。

帧格式：

```
[addr] FD [dir] [speed_H] [speed_L] [acc] [pulse_31_24] [pulse_23_16] [pulse_15_8] [pulse_7_0] [mode] [sync] 6B
```

参数：

- `dir`：`00=CW`，`01=CCW`。
- `speed`：2 字节 RPM，高字节在前。
- `acc`：加速度档位。
- `pulse`：4 字节脉冲数，高字节在前。
- `mode = 0x00`：相对位置模式。
- `mode = 0x01`：绝对位置模式。
- `sync`：多机同步标志。

示例：

```
01 FD 01 05 DC 00 00 00 7D 00 00 00 6B
```

含义：

- 地址 `01`。
- 方向 `CCW`。
- 速度 `1500 RPM`。
- 加速度 `0`。
- 脉冲数 `0x00007D00 = 32000`。
- 相对位置模式。
- 不启用同步。

在 16 细分下，`32000` 脉冲 = `10` 圈 = `3600°`。

成功返回：

```
01 FD 02 6B
```

### 4. 立即停止

功能：让电机立即停止，可用于速度模式和位置模式。

帧格式：

```
[addr] FE 98 [sync] 6B
```

示例：

```
01 FE 98 00 6B
```

成功返回：

```
01 FE 02 6B
```

### 5. 多机同步运动

功能：让之前带 `sync=01` 的多个电机命令同时开始执行。

帧格式：

```
[addr] FF 66 6B
```

建议用广播地址：

```
00 FF 66 6B
```

多机同步流程：

1. 给 1 号电机发送运动命令，最后 `sync=01`，此时电机不动。
2. 给 2 号电机发送运动命令，最后 `sync=01`，此时电机不动。
3. 发送 `00 FF 66 6B`，所有缓存了同步命令的电机同时启动。

资料示例：

```
01 FD 01 05 DC 08 00 00 7D 00 00 01 6B
02 FD 00 03 E8 0A 00 00 FA 00 01 01 6B
00 FF 66 6B
```


六、回零相关命令
----------------

### 1. 设置单圈回零零点

功能：把当前机械位置设为单圈回零的零点。

帧格式：

```
[addr] 93 88 [save] 6B
```

示例：

```
01 93 88 01 6B
```

含义：把 1 号电机当前所在位置保存为单圈回零零点。

成功返回：

```
01 93 02 6B
```

### 2. 触发回零

功能：执行回零动作。

帧格式：

```
[addr] 9A [origin_mode] [sync] 6B
```

回零模式：

- `0x00`：单圈就近回零。
- `0x01`：单圈方向回零。
- `0x02`：多圈无限位碰撞回零。
- `0x03`：多圈有限位开关回零。

示例：

```
01 9A 00 00 6B
```

含义：1 号电机立即执行单圈就近回零。

成功返回：

```
01 9A 02 6B
```

条件不满足返回：

```
01 9A E2 6B
```

常见原因：堵转保护、电机未使能、单圈回零零点无效。

### 3. 强制中断并退出回零

帧格式：

```
[addr] 9C 48 6B
```

示例：

```
01 9C 48 6B
```

成功返回：

```
01 9C 02 6B
```

### 4. 读取回零状态标志位

帧格式：

```
[addr] 3B 6B
```

示例：

```
01 3B 6B
```

返回：

```
[addr] 3B [flags] 6B
```

`flags` 位定义：

- `flags & 0x01`：编码器就绪。
- `flags & 0x02`：校准表就绪。
- `flags & 0x04`：正在回零。
- `flags & 0x08`：回零失败。


七、触发动作命令
----------------

### 1. 触发编码器校准

帧格式：

```
[addr] 06 45 6B
```

示例：

```
01 06 45 6B
```

成功返回：

```
01 06 02 6B
```

注意：

- 第一次上电若提示 `Not Cal`，需要先校准。
- 校准前尽量空载。
- 条件不满足可能是当前为开环模式或触发了堵转保护。

### 2. 将当前位置角度清零

功能：清零当前位置角度、位置误差、脉冲数等。

帧格式：

```
[addr] 0A 6D 6B
```

示例：

```
01 0A 6D 6B
```

成功返回：

```
01 0A 02 6B
```

### 3. 解除堵转保护

帧格式：

```
[addr] 0E 52 6B
```

示例：

```
01 0E 52 6B
```

成功返回：

```
01 0E 02 6B
```

条件不满足返回：

```
01 0E E2 6B
```

常见原因：当前没有触发堵转保护。

### 4. 恢复出厂设置

不建议在比赛程序里调用，除非明确需要重新初始化驱动。

帧格式：

```
[addr] 0F 5F 6B
```

示例：

```
01 0F 5F 6B
```

成功返回：

```
01 0F 02 6B
```

注意：恢复出厂设置后需要断电重启，并重新空载校准编码器。


八、常用读取命令
----------------

读取命令一般格式短，返回数据长度按功能不同变化。
解析时先检查：

1. 返回第 0 字节是否为目标地址。
2. 返回第 1 字节是否为对应功能码。
3. 最后一字节是否为 `0x6B`。
4. 若返回 `[addr] 00 EE 6B`，表示命令错误。

### 1. 读取固件和硬件版本

发送：

```
01 1F 6B
```

返回：

```
[addr] 1F [firmware] [hardware] 6B
```

### 2. 读取总线电压

发送：

```
01 24 6B
```

返回：

```
[addr] 24 [voltage_H] [voltage_L] 6B
```

解析：

```
voltage_mV = (voltage_H << 8) | voltage_L
```

### 3. 读取相电流

发送：

```
01 27 6B
```

返回：

```
[addr] 27 [current_H] [current_L] 6B
```

解析：

```
current_mA = (current_H << 8) | current_L
```

### 4. 读取电机实时转速

发送：

```
01 35 6B
```

返回：

```
[addr] 35 [sign] [speed_H] [speed_L] 6B
```

解析：

- `sign = 0x00`：正数。
- `sign = 0x01`：负数。
- `speed_rpm = (speed_H << 8) | speed_L`。
- 若 `sign == 0x01`，最终速度取负。

示例返回：

```
01 35 01 05 DC 6B
```

表示 `-1500 RPM`。

### 5. 读取电机实时位置

发送：

```
01 36 6B
```

返回：

```
[addr] 36 [sign] [pos_31_24] [pos_23_16] [pos_15_8] [pos_7_0] 6B
```

解析：

```
raw = (pos_31_24 << 24) | (pos_23_16 << 16) | (pos_15_8 << 8) | pos_7_0
angle_deg = raw * 360.0 / 65536.0
if sign == 0x01:
    angle_deg = -angle_deg
```

说明：位置返回值中 `65536` 表示一圈。

### 6. 读取电机目标位置

发送：

```
01 33 6B
```

返回和解析方式同“读取电机实时位置”。

### 7. 读取电机位置误差

发送：

```
01 37 6B
```

返回和解析方式同“读取电机实时位置”。

### 8. 读取电机状态标志位

发送：

```
01 3A 6B
```

返回：

```
[addr] 3A [flags] 6B
```

`flags` 位定义：

- `flags & 0x01`：电机已使能。
- `flags & 0x02`：电机已到位。
- `flags & 0x04`：电机堵转。
- `flags & 0x08`：已触发堵转保护。

建议：

- 位置模式后可轮询 `3A`，当 `flags & 0x02` 为真时认为到位。
- 如果 `flags & 0x08` 为真，先发 `0E 52` 解除堵转保护，再重新使能/运动。

### 9. 读取驱动配置参数

发送：

```
01 42 6C 6B
```

返回：

```
[addr] 42 [config...] 6B
```

常用解析项：

- 串口波特率配置。
- ID 地址。
- 校验方式。
- 控制命令应答模式。
- 堵转保护参数。
- 位置到达窗口。

如果只是实现比赛控制，通常不必解析完整配置。

### 10. 读取系统状态参数

发送：

```
01 43 7A 6B
```

返回：

```
[addr] 43 [system_status...] 6B
```

包含总线电压、相电流、编码器值、目标位置、实时转速、实时位置、位置误差、就绪标志、电机状态标志等。

如果 MCU 资源紧张，建议用单独读取命令代替这个大包。


九、常用配置命令
----------------

配置命令一般会修改驱动参数。比赛程序中只保留确实需要的配置，避免误改驱动。

### 1. 修改任意细分

帧格式：

```
[addr] 84 8A [save] [microstep] 6B
```

说明：

- `microstep = 0x00` 表示 256 细分。
- 其他值直接表示细分数，例如 `0x10 = 16`。

示例：

```
01 84 8A 01 10 6B    // 保存为16细分
```

成功返回：

```
01 84 02 6B
```

### 2. 修改 ID 地址

帧格式：

```
[addr] AE 4B [save] [new_id] 6B
```

示例：

```
01 AE 4B 01 10 6B    // 把当前1号电机改为16号并保存
```

成功返回：

```
01 AE 02 6B
```

### 3. 切换开环/闭环模式

帧格式：

```
[addr] 46 69 [save] [mode] 6B
```

参数：

- `mode = 0x01`：开环模式。
- `mode = 0x02`：闭环 FOC 模式。

示例：

```
01 46 69 01 02 6B    // 保存为闭环模式
```

### 4. 修改开环模式工作电流

帧格式：

```
[addr] 44 33 [save] [current_H] [current_L] 6B
```

示例：

```
01 44 33 00 03 E8 6B    // 临时设为1000mA，不保存
```

### 5. 存储一组上电自动运行的速度模式参数

功能：让驱动器上电后自动按某个速度运行。
比赛程序一般不建议启用，除非确实要脱离 MCU 自动运行。

帧格式：

```
[addr] F7 1C [store_or_clear] [dir] [speed_H] [speed_L] [acc] [en_pin_control] 6B
```

参数：

- `store_or_clear = 0x01`：存储。
- `dir`：`00=CW`，`01=CCW`。
- `speed`：2 字节 RPM。
- `acc`：加速度。
- `en_pin_control = 0x01`：允许 En 引脚控制启停；`0x00`：不启用。

示例：

```
01 F7 1C 01 01 05 DC 0A 01 6B
```

含义：保存为上电后以 CCW、1500 RPM、加速度 10 自动运行，并允许 En 控制启停。

### 6. 修改通讯输入速度是否缩小 10 倍

帧格式：

```
[addr] 4F 71 [save] [enable] 6B
```

参数：

- `enable = 0x00`：不缩小，默认 1 RPM 分辨率。
- `enable = 0x01`：缩小 10 倍，发送 1 表示实际 0.1 RPM。

示例：

```
01 4F 71 01 01 6B    // 保存启用0.1RPM输入
```

影响范围：

- 速度模式控制。
- 位置模式控制。
- 存储速度模式参数。


十、建议给 STM32 工程实现的 API
-------------------------------

建议新建或完善步进电机驱动模块，例如：

```
User/Motor/bujin_step.h
User/Motor/bujin_step.c
```

可提供如下函数，内部只负责拼帧和调用 HAL UART 发送：

```c
void Bujin_Enable(uint8_t addr, uint8_t enable);
void Bujin_SetSpeed(uint8_t addr, uint8_t dir, uint16_t rpm, uint8_t acc);
void Bujin_MovePulse(uint8_t addr, uint8_t dir, uint16_t rpm, uint8_t acc,
                     uint32_t pulse, uint8_t absolute);
void Bujin_MoveAngle(uint8_t addr, uint8_t dir, uint16_t rpm, uint8_t acc,
                     float angle_deg, uint8_t absolute, uint16_t microstep);
void Bujin_Stop(uint8_t addr);
void Bujin_SyncStart(void);
void Bujin_ClearPosition(uint8_t addr);
void Bujin_ReleaseStall(uint8_t addr);
void Bujin_SetOrigin(uint8_t addr, uint8_t save);
void Bujin_GoOrigin(uint8_t addr, uint8_t origin_mode);
void Bujin_ReadSpeed(uint8_t addr);
void Bujin_ReadPosition(uint8_t addr);
void Bujin_ReadStatus(uint8_t addr);
```

实现注意：

1. 所有命令数组最后一个字节默认填 `0x6B`。
2. `uint16_t`、`uint32_t` 发送时手动拆成高字节在前。
3. `MoveAngle` 内部按 `pulse = angle_deg / 360.0f * (200 * microstep)` 转换。
4. 如果位置模式传入负角度，也可以由上层转成方向 `dir` + 正脉冲数，避免协议层处理有符号数。
5. 发送动作命令后建议等待并解析返回，至少判断是否收到 `[addr] [func] 02 6B`。
6. 若收到 `[addr] [func] E2 6B`，先读 `3A` 状态，判断是否未使能或堵转保护。
7. 发送给广播地址 `0x00` 时，多个电机会同时返回，串口总线可能冲突；广播更适合只发不等回复的同步触发。


十一、最小可用控制流程
----------------------

### 1. 单电机连续转动

```
01 F3 AB 01 00 6B              // 使能
01 F6 00 01 F4 0A 00 6B        // CW, 500RPM, acc=10
01 FE 98 00 6B                 // 立即停止
```

### 2. 单电机相对转一圈，16细分

```
01 F3 AB 01 00 6B              // 使能
01 FD 00 01 F4 0A 00 00 0C 80 00 00 6B
```

解释：

- `0x01F4 = 500 RPM`
- `0x00000C80 = 3200` 脉冲
- `mode=00` 相对位置
- `sync=00` 立即执行

### 3. 读状态判断是否到位

```
01 3A 6B
```

若返回：

```
01 3A 03 6B
```

则：

- `0x03 & 0x01`：已使能。
- `0x03 & 0x02`：已到位。

### 4. 堵转保护恢复

```
01 3A 6B                       // 读取状态
01 0E 52 6B                    // 解除堵转保护
01 F3 AB 01 00 6B              // 必要时重新使能
```


十二、常见硬件/屏幕提示
-----------------------

- `Not Cal`：未校准编码器，需执行 Cal 或发送 `01 06 45 6B`。
- `Waiting V+ Power`：V+ 和 GND 电压不足，建议 12-32V，常用 24V。
- `Phase A+ A- Error` / `Phase B+ B- Error` / `Phase AA BB Error`：电机线序错误。
- `Magnet Loss`：未检测到磁铁，检查磁铁和驱动板安装。
- `Go to Origin Fail`：回零失败，检查力矩、速度、阈值、限位/机械结构。

# 第一问难点分析

## 难点清单

| 序号 | 难点 | 核心矛盾 | 解决方向 |
|---|---|---|---|
| 1 | 高速行驶 | 15 秒跑 400cm，平均 ≥27cm/s | 分段调速：直道冲刺，拐前降速 |
| 2 | 直角转弯 | 正方形 90° 拐角，PID 纯巡线过不去 | LeftRoad 检测触发 + 陀螺仪闭环 pivot |
| 3 | 精确停车 | 检测到拐角 A 再刹车，物理窗口仅 ~4cm | 编码器测距预判，不等拐角检测 |
| 4 | 起步即拐角 | 发车位置就是 A 点，传感器上来就看到 LeftRoad | START 状态特判，cross_cnt 预设为 1 |
| 5 | 左转陷阱 | 回到 A 时 LeftRoad 触发左转而非停车 | cross_cnt > 3 时覆盖 LeftRoad 行为为 STOP |

---

## 难点 1：高速行驶

**问题**：15 秒跑 400cm 正方形，平均速度 ≥ 27cm/s。但拐角处不能高速。

**方案**：分段速度策略

```
每条 100cm 边：
  前 80cm → base_speed 高速（30-35）
  后 20cm → base_speed 低速（10-12）
  编码器里程触发速度切换
```


---

## 难点 2：直角转弯

**问题**：正方形拐角是尖锐 90°，线在拐角处有盲区。PID 纯巡线在盲区里 `diff` 崩溃，车要么冲出赛道要么方向歪掉。

**方案**：保留 `get_road_type()` 的 LeftRoad 作为触发器，改进执行方式

```
检测到 LeftRoad
  → 记录当前陀螺仪角度
  → 目标角度 = 当前 - 90°（逆时针左转）
  → 陀螺仪 PID 闭环转 90°
  → 误差 < 2° 持续 N 帧 → 转弯完成
  → 切回巡线
```
注意 由于转弯需要迅速 角度环的PID可能需要激进一点 KP和KI都得给大

**拐前预减速**：编码器走到 ~80cm 降速，确保 pivot 时速度足够低。降速太早浪费时间，太晚刹不住，80cm 是初始值需实测校准。

---

## 难点 3：精确停车

**问题**：灰度传感器检测到拐角 A 时，车头已在 AD 线内。剩余制动距离 = 线宽 1.8 + 允许 3 - 传感器距前沿 0.7 ≈ 4.1cm。即使低速（12cm/s），刹车延迟 + 惯性容易超限。

**方案**：编码器测距做主逻辑，灰度检测做兜底

```
拐角 4（B 点）转弯完成
  → 清零编码器里程
  → base_speed = 12（低速精确）
  → 每周期累加里程
  → 里程 >= STOP_TARGET（实测校准，约 97cm）
  → 立即 STOP

同时全程：
  灰度检测到 LeftRoad → STOP（兜底，精度差但不会完全错过）
  蓝牙收到 'S' → STOP（人工核武器  我他妈手机直接手动停车 骗骗你的呀）
```

**为什么编码器方案可行**：车全程在巡线，黑线锁死横向偏差，编码器只管纵向距离。低速下轮子打滑极小，100cm 累积误差 1-2cm。赛前实测校准（手动放车到理想停车位，读编码器值）可进一步吸收系统误差。


## 难点 4：起步即拐角

**问题**：发车时车站在 A 点上（车身在 AB，前沿在 AD）。踩油门不到 1cm，传感器就扫到 LeftRoad。如果正常响应，cross_cnt 从 0 跳 1，后续所有拐角计数全部错位。

**方案**：我没想好
---

## 难点 5：左转陷阱

**问题**：拐角 4 之后车在 BA 边上朝左走，到 A 时灰度检测到 LeftRoad。`follow_line()` 里 LeftRoad 分支是做**左转 pivot**的，车会在 A 点原地转一圈才停。

**方案**：cross_cnt 状态覆盖转弯行为

```
if (检测到 LeftRoad):
    if (cross_cnt >= 4):
        motion = STOP    // 最后一个拐角：停车，不转弯
    else:
        执行左转 pivot   // 正常拐角：转弯
```

---

## 蓝牙核武器

STM32 一个 UART 接蓝牙模块，独立于自动逻辑：

```
UART 收到 'S' → 直接设 motion = STOP
```

不做状态机耦合。同时绑定 `button_B11` 作为有线版核武器。用途是自动停车失败时的最后一刻人工干预。

# 2026年全国大学生电子设计竞赛 C题：多功能小车 —— 赛题解析与项目需求

> 本文件详细解析赛题要求，明确小车需要完成的所有任务、场地约束、评分标准及实现策略。

---

## 一、赛题概述

设计并制作一辆**多功能智能小车**，使其能够在规定的正方形轨迹上完成自动循迹行驶、负重运行、干扰抵抗、图形测量与识别、无线通信、精确定位停车等功能。

---

## 二、场地与硬件约束

### 2.1 行驶轨迹（地图）

- **形状**：外边长 **100cm × 100cm** 的正方形
- **边线**：黑色电工胶带，线宽 **1.8cm ± 0.2cm**
- **顶点命名**：按逆时针顺序定义为 **A、B、C、D** 四个顶点
- **发车点**：
  - **AB发车点**：小车前沿中心点投影压在 **AD线外沿** 上，小车朝 **B点方向**（逆时针行驶）
  - **AD发车点**：小车前沿中心点投影压在 **AB线外沿** 上，小车朝 **D点方向**（顺时针行驶）
- **示意图**：

```
        D ━━━━━━━━━━━━━━━ C
        ┃               ┃
        ┃   100cm       ┃
        ┃               ┃
        A ━━━━━━━━━━━━━━━ B
```

### 2.2 小车机械约束

| 约束项 | 要求 |
|--------|------|
| 尺寸 | ≤ 20cm(长) × 15cm(宽) × 25cm(高)，**含所有安装模块** |
| 轮式 | **3~4个轮式**，不得使用履带、麦氏轮 |
| 动力 | 车载电池供电，行驶中**严禁人为干涉、遥控** |
| 负重 | 需能承载 **330ml 矿泉水**（约330g）完成指定动作 |
| 投影 | 行驶过程中小车投影**必须始终在黑线上**，完全脱离即判该次失败 |
| 电池 | 进入测试环节后中途**不得更换电池** |

### 2.3 辅助道具

#### （1）干扰 A4 纸（基本要求2使用）
- **位置**：正方形 **BC 边中心部位**，横向贴放
- **要求**：纵向黑色线与原 BC 轨迹黑线重合
- **类型**：共 **3 种**（自备，测评时随机选一种）
- **制作建议**：A4白纸 + 黑色电工胶带
- **尺寸示例**：7.5cm×1.8cm、16.8cm×1.8cm 等组合

#### （2）待测 A4 纸（发挥部分使用）
- **位置**：正方形 **CD 边中心部位**，横向贴放
- **要求**：纵向黑色线与原轨迹黑线重合
- **类型**：共 **4 种** —— **圆形、正方形、三角形、椭圆形**
- **尺寸**：l1、l2 参数范围 **1cm ~ 5cm**（由图形内部黑色标记定义）
- **测试次数**：两次，一次自备，一次测评人员提供

## 三、任务要求详解

### 3.1 基本要求（共50分）

#### 任务（1）：循迹走1圈，停回发车点 【20分】

| 项目 | 要求 |
|------|------|
| 发车点 | AB 发车点 |
| 行驶方向 | **逆时针** 行驶 |
| 行驶路径 | 沿正方形黑线完整走 **1 圈** |
| 声光提示 | **启动和停车时** 均需有声光提示 |
| 停车精度 | 停车时小车前沿中心点与 **AD 线外沿** 距离 **D1 ≤ 3cm** |
| 时间限制 | **t1 ≤ 15 秒** |

**核心难点**：
- 可靠识别4个路口（十字交叉），准确计数
- 第4个路口后减速，精确对准起点线
- 15秒内完成400cm路程，平均速度需 ≥ 27cm/s

---

#### 任务（2）：3/4圈 + 掉头返回发车点 【30分】

| 项目 | 要求 |
|------|------|
| 发车点 | AB 或 AD（由测评人员现场选择） |
| 行驶路径 | 沿布置有**干扰A4纸**（BC边）的轨迹行驶 **3/4 圈**，然后**掉头原路返回** |
| 声光提示 | 启动和停车时均需有声光提示 |
| 停车精度 | 停车时小车前沿中心点与**终点线外沿**距离 **D2 ≤ 3cm** |
| 时间限制 | **t2 ≤ 25 秒** |

**关键动作分解**：
```
AB发车(逆时针) → A→B→C(到达3/4圈) → 掉头 → C→B→A(返回)
AD发车(顺时针) → A→D→C(到达3/4圈) → 掉头 → C→D→A(返回)
```

**核心难点**：
- **干扰纸处理**：BC边中心的干扰纸会形成"假路口"，需结合**编码器里程**或**位置记忆**进行屏蔽
- **掉头动作**：走3/4圈后需要**原地掉头180°**或**弧形掉头**，然后反向循迹
- **反向行驶**：掉头后传感器前后颠倒，需处理左右反转逻辑（或采用对称安装）
- **时间充裕度**：25秒完成约600cm路程（去300cm + 回300cm），平均速度要求较低

---

### 3.2 发挥部分（共50分）

#### 任务（3）：负重 + 测量A4纸参数 + 无线传输 【30分】

| 项目 | 要求 |
|------|------|
| 负重 | 载有 **330ml 矿泉水** |
| 发车点 | AB 或 AD（测评人员选择） |
| 检测物 | CD 边中心贴放的**待测A4纸** |
| 测量参数 | A4纸上的 **l1、l2** 尺寸（1cm ≤ l1, l2 ≤ 5cm） |
| 传输要求 | 通过**无线传输**至**手机显示**，相对误差 **≤ 20%** |
| 行驶路径 | 沿轨迹行驶 **1 圈** 后停回发车点 |
| 声光提示 | 启动和停车时均需有声光提示 |
| 停车精度 | **D3 ≤ 3cm** |
| 时间限制 | **t3 ≤ 15 秒** |

**核心难点与技术方案**：
- **视觉测量**：需用 **MaixCam** 识别A4纸并测量内部标记尺寸
  - 标定：利用A4纸已知尺寸（纵向18cm × 横向1.8cm黑线）建立像素-厘米映射
  - 需在MaixCam端运行图像处理脚本（MicroPython），将l1、l2转为像素长度
- **无线通信**：STM32 → 蓝牙模块（如HC-05/06）→ 手机串口助手APP
- **负重影响**：330g额外负载会改变电机响应特性，PID参数必须在负重状态下整定

---

#### 任务（4）：负重 + 图形识别 + 精确定位停车 【20分】

| 项目 | 要求 |
|------|------|
| 负重 | 载有 **330ml 矿泉水** |
| 发车点 | **AB 发车点** |
| 检测物 | CD 边贴放的**待测A4纸**（4种类型之一） |
| 识别要求 | 判断 A4 纸类型：**圆形、正方形、三角形、椭圆形** |
| 停车位置 | 行驶轨迹图**对角线 AC 和 BD 的三等分点**对应图片位置 |
| 停车要求 | 共 **4 个点位**，图形任选点位**平铺放置**（不重复），停车时小车**外轮廓压上对应图形即可** |
| 时间限制 | **t4 ≤ 30 秒** |

**关键几何计算**：
- 正方形对角线长度 = 100√2 ≈ **141.4 cm**
- 三等分点距顶点距离 = 141.4 / 3 ≈ **47.1 cm**
- 从CD边中心到各三等分点需要**沿对角线方向**行驶约47cm

**核心难点与技术方案**：
- **图形识别**：MaixCam拍摄CD边A4纸，通过形状特征判断类型
- **对角线导航**：对角线方向**无黑线**，必须采用**开环+陀螺仪闭环**方式
  - 从CD边中点出发，先沿CD走到C或D
  - 转弯 **45°**（由陀螺仪精确控制）
  - 沿对角线方向直线行驶 **47cm**（编码器里程计）
  - 到达后停车，外轮廓压上图形即可（允许一定误差）
- **无IMU寸步难行**：没有陀螺仪的话，45°转弯纯靠编码器差速极不可靠

## 五、总体实现策略

### 5.1 分层架构

```
┌─────────────────────────────────────────┐
│  应用层：任务调度 (TASK)                  │
│  - 模式切换：走圈 / 掉头 / 测量 / 定位    │
├─────────────────────────────────────────┤
│  行为层：状态机 (MOTION_STATION)          │
│  - STOP / FIND_LINE / KEEP_ANGLE          │
├─────────────────────────────────────────┤
│  控制层：闭环控制                         │
│  - 循迹PID（灰度传感器偏差）              │
│  - 速度PID（编码器反馈）                  │
│  - 角度PID（陀螺仪yaw角反馈）             │
├─────────────────────────────────────────┤
│  感知层：传感器融合                       │
│  - GW_ANALOGUE（循迹+路口）               │
│  - GY901（航向角）                        │
│  - 编码器（里程计）                       │
│  - MaixCam（视觉识别+测距）               │
├─────────────────────────────────────────┤
│  驱动层：硬件抽象                         │
│  - 电机PWM / 舵机PWM / LED / 蜂鸣器       │
│  - ADC / I2C / UART / DMA                 │
└─────────────────────────────────────────┘
```

### 5.2 关键算法链

**（1）基础循迹**
```
灰度传感器 → 归一化 → diff偏差(-42~+42) → PID → 左右轮差速 → 电机驱动
```

**（2）路口处理**
```
数字量8bit → 积分滤波 → 路口类型判定(十字/T字/左弯/右弯) → 圈数计数 → 动作触发
```

**（3）角度保持（掉头/对角线）**
```
目标角度 - 当前yaw角 → 角度差归一化(-180~+180) → PID → 左右轮差速转向
```

**（4）里程定位**
```
编码器脉冲 → 轮速 → 积分 → 行驶距离 → 配合路口清零 → 分段定位
```

### 5.3 推荐开发顺序

1. **Week 1**：基础循迹调稳 + 路口识别准确率测试（重点测试干扰纸场景）
2. **Week 2**：走1圈计数与精确停车（任务1）
3. **Week 3**：3/4圈 + 掉头返回（任务2）+ 干扰纸滤波算法
4. **Week 4**：MaixCam视觉程序开发 + 蓝牙联调（任务3通信部分）
5. **Week 5**：图形识别 + 对角线三等分点定位（任务4）
6. **Week 6**：负重全流程联调 + 鲁棒性优化

## 六、必须攻克的技术难点

| 难点 | 影响 | 解决思路 |
|------|------|----------|
| **干扰A4纸误判路口** | 任务2失败 | 编码器里程区域屏蔽（BC段40~60cm不响应路口） |
| **停车精度≤3cm** | 任务1/2/3扣分 | 末端减速 + 传感器对准（中间两路压线） |
| **对角线无黑线导航** | 任务4无法完成 | 陀螺仪角度保持 + 编码器开环里程（约47cm） |
| **负重电机特性变化** | 所有任务 | 提高基础PWM、加大积分限幅、带载调参 |
| **15秒时间限制** | 任务1/3时间紧 | 直线段高速(40cm/s)、弯道适当减速(25cm/s) |
| **图形识别准确率** | 任务4核心 | MaixCam端跑 shape detection，多帧投票防误判 |

---

> **结论**：本项目是一个典型的"感知-决策-控制"一体化嵌入式系统项目。基础循迹和圈数计数决定基本分（50分），而**陀螺仪精确定位**和**MaixCam视觉扩展**是冲击发挥分（50分）的核心技术壁垒。
